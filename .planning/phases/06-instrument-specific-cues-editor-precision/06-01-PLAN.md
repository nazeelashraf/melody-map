## Spec Requirements (CUE-01 through CUE-05)

1. **CUE-01**: Each lyric line stores independent cue data for piano and guitar instead of one shared chord line
2. **CUE-02**: Guitar cue text can differ from piano cue text on the same lyric line
3. **CUE-03**: Percussion cues edited as five named lanes (C, H, R, S, B) aligned to lyric line
4. **CUE-04**: Multiple percussion hits can stack at same lyric position across lanes
5. **CUE-05**: Copy-to-all and copy-to-one actions per lyric line for cue data

## Spec Requirements (EDIT-01 through EDIT-03)

6. **EDIT-01**: Multi-character chord entry (e.g., "Bm") keeps caret stable without jumping
7. **EDIT-02**: Vertically linked monospace cue and lyric fields for reliable position targeting
8. **EDIT-03**: Cue rows visually distinct from lyrics without disabled appearance

## Spec Requirements (PERF-01 through PERF-03)

9. **PERF-01**: Performance view shows cues for selected instrument only
10. **PERF-02**: Piano/guitar cues render in distinct monospace style separated from lyrics
11. **PERF-03**: Percussion renders as separate C, H, R, S, B lanes aligned to lyric line

## Implementation Decisions (from 06-CONTEXT.md)

- **D-01**: SheetEditor edits one instrument at a time per lyric line
- **D-04**: Percussion in edit view shows one stacked block with all 5 lanes
- **D-05**: Performance view uses compact lane letters (C, H, R, S, B) with visible legend
- **D-07**: Copy actions live in the lyric-line action row
- **D-08**: Percussion copy operations treat 5-lane set as one bundled unit
- **D-09**: Copy-to-one uses inline target buttons (not menus)
- **D-10**: Piano/guitar cue editing feels like tight aligned text rows (not per-cell grid)
- **D-11**: Initial cue positioning via clicking cue row OR clicking lyric row to mirror caret
- **D-12**: Subtle position guide/highlight for alignment tracking (lightweight, no ruler)
