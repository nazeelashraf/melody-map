**src/styles/print.css — New file:**
Create a print media stylesheet:
```css
@media print {
  /* Hide all interactive chrome */
  button,
  input,
  textarea,
  a,
  nav,
  [role="navigation"] {
    display: none !important;
  }

  /* Remove background gradients and shadows */
  body {
    background: white !important;
  }

  /* Remove shadows, borders, backdrop filters from panels */
  div[style*="box-shadow"],
  div[style*="backdrop-filter"],
  div[style*="background: linear-gradient"] {
    box-shadow: none !important;
    backdrop-filter: none !important;
    background: white !important;
  }

  /* Show sheet content clearly */
  pre {
    background: white !important;
    color: black !important;
    border: 1px solid #ccc !important;
    page-break-inside: avoid;
  }

  /* Ensure readable fonts */
  body, h1, h2, h3, p, span, pre {
    color: black !important;
  }

  /* Page break before each major section */
  section {
    page-break-inside: avoid;
  }

  /* Hide the library home — only print sheet/composition editor pages */
  /* The SheetList page should not print well (it's a management page) */
}
```

Note: Since the app uses inline styles (not CSS classes), the print.css selectors target HTML elements. This is the pragmatic approach given the project's architecture. The key rules hide buttons/inputs/links and ensure content is readable.
